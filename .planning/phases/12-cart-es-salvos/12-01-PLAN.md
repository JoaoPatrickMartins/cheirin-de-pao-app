<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| client→API (saved-cards) | Usuário autenticado acessa/modifica seus próprios cartões — nunca cartões de outro usuário |
| API→MP Customer API | access_token nunca exposto ao frontend; CVV (security_code) processado apenas no backend e descartado imediatamente após CardToken.create |
| frontend→API (payments/card) | savedCardId enviado pelo cliente — backend valida ownership antes de usar |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Tampering (IDOR) | GET/PATCH/DELETE /users/me/cards/:id | mitigate | Verificar savedCard.userId === request.user.id antes de qualquer operação; retornar 404 (nunca 403 — não revela existência do recurso) |
| T-12-02 | Information Disclosure | CVV no payload de POST /payments/card | mitigate | security_code nunca logado, nunca persistido, nunca retornado; CardToken.create descarta após uso único; enforçar no código e nos testes |
| T-12-03 | Tampering | Limite de 3 cartões via chamadas paralelas | mitigate | Verificar count no service com prisma.savedCard.count antes de chamar Customer.createCard no MP — validação server-side, não apenas frontend |
| T-12-04 | Tampering | Salvar cartão sem pagamento (abuso de API) | mitigate | Customer.createCard e prisma.savedCard.create chamados somente dentro do fluxo de createCard() após Payment.create bem-sucedido |
| T-12-05 | Tampering | setDefault race condition (2 padrões simultâneos) | mitigate | prisma.$transaction([updateMany isDefault:false, update isDefault:true]) — operação atômica |
| T-12-06 | Denial of Service | Criação ilimitada de MP Customers para 1 usuário | mitigate | getOrCreateMpCustomer verifica user.mpCustomerId primeiro; Customer.search por email antes de criar; nunca criar se mpCustomerId já existe |
| T-12-SC | Tampering | npm/pip/cargo installs | accept | Nenhum novo pacote instalado nesta fase — mercadopago@3.1.0 e @mercadopago/sdk-react@1.0.7 já auditados em uso desde Phase 3 |
</threat_model>
