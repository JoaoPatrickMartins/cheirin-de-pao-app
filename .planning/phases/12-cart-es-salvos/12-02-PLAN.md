<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| browser→API | CVV digitado no input nativo nunca sai do componente exceto como securityCode no body do POST — não persistido em localStorage nem em state além da sessão atual |
| browser→MP Brick | Dados de cartão novo passam pelo Brick (PCI DSS scope do MP) — frontend nunca vê número de cartão completo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-07 | Information Disclosure | CVV no estado React (cvv string) | mitigate | setCvv('') imediatamente após POST disparado; estado cvv nunca persistido fora do componente |
| T-12-08 | Tampering | selectedCardId manipulado no frontend | accept | Backend valida ownership (savedCard.userId === userId) — frontend só envia o ID, validação real é no backend (T-12-01 do plano 01) |
| T-12-09 | Tampering | saveCard flag falso no body | accept | Backend impõe limite de 3 cartões server-side (T-12-03 do plano 01) — frontend só indica intenção |
| T-12-SC | Tampering | npm/pip/cargo installs | accept | Nenhum pacote novo — apenas modificação de código existente |
</threat_model>
