## Screens e Layout

### CardPaymentScreen modificada (`/client/creditos/cartao`)

**Estrutura atual:** Tela com AppBar + MP Bricks `<CardPayment>` direto.

**Nova estrutura (Phase 12):** A tela passa a ter dois modos distintos, determinados pela presença de cartões salvos do usuário.

#### Modo A — Cliente tem cartões salvos

```
AppBar: ← "Pagamento com cartão"
Texto segurança: "Pagamento processado com segurança pelo Mercado Pago"
─────────────────────
Seção: "Seus cartões"
  Lista vertical de cards de cartão salvo (até 3)
  Cada card: bandeira | •••• {4 dígitos} | vencimento {MM/YY} | radio indicator
  → O último cartão usado (ou o padrão) vem pré-selecionado
─────────────────────
Separador "ou" (linha horizontal com texto centralizado, 12.5px, color-text-ter)
─────────────────────
Card — "Adicionar novo cartão" (tap para expandir o Brick)
  Quando expandido: exibe MP Bricks CardPayment inline
  Checkbox: "Salvar para compras futuras" (checked por padrão)
─────────────────────
CTA bar fixa acima do tab bar:
  Botão primário: "Pagar com este cartão" (quando cartão salvo selecionado)
  OU: "Salvar cartão e pagar" (quando Brick expandido + checkbox marcado)
  OU: "Pagar sem salvar" (quando Brick expandido + checkbox desmarcado, menor destaque visual)
```

**Hierarquia visual no modo A:**
- Cartões salvos: destaque principal — cards com radio indicator, borda accent no selecionado
- "Adicionar novo cartão": destaque secundário — card com ícone `plus`, fundo `--color-surface`, borda `--color-border`
- "Usar sem salvar": menor destaque — texto 12.5px (`--text-sm`), cor `--color-text-sec`, sem fundo de botão

#### Modo B — Cliente não tem cartões salvos (fluxo atual sem alteração estrutural)

```
AppBar: ← "Pagamento com cartão"
Texto segurança
MP Bricks CardPayment (comportamento atual preservado)
Checkbox: "Salvar este cartão para compras futuras" (novo — unchecked por padrão)
```

**Nota CARD-03:** A opção de compra sem salvar deve ter menor destaque visual em ambos os modos. No Modo A, o botão de menor destaque é "Pagar sem salvar". No Modo B, o checkbox desmarcado já expressa visualmente a opção de não salvar.

**Nota D-16 (STATE.md):** CVV obrigatório via Brick a cada transação com cartão salvo. O fluxo de cartão salvo não bypassa o Brick — o Brick é configurado no modo "CVV only" via MP Customer API para capturar apenas o CVV e gerar um novo token seguro por transação.

## Fluxo Completo de Interação

### Fluxo A — Pagar com cartão salvo

1. Cliente toca em "Cartão" no seletor de método de pagamento na `CombosScreen`
2. Navega para `CardPaymentScreen` (location.state com `{ comboId?, customQuantity?, amount }`)
3. Tela carrega lista de cartões via `GET /users/me/cards`
4. Lista exibida com o cartão padrão pré-selecionado (radio filled)
5. Cliente confirma seleção → toca em "Pagar com este cartão"
6. CTA muda para "Processando..." (disabled)
7. Backend: `POST /payments/card` com `{ savedCardId, comboId?, customQuantity? }`
8. Sucesso: navegar para `/client/creditos/sucesso` (padrão existente)
9. Erro: toast de erro, CTA volta ao estado inicial

### Fluxo B — Cadastrar novo cartão com opção de salvar

1. Cliente toca em "Adicionar novo cartão" (card colapsado expande)
2. MP Bricks `<CardPayment>` renderiza dentro do card
3. Checkbox "Salvar para compras futuras" exibido abaixo do Brick (checked por padrão)
4. CTA muda para "Salvar cartão e pagar"
5. Cliente preenche dados no Brick e toca no CTA
6. `onSubmit` do Brick recebe formData (token, payment_method_id, etc.)
7. Backend: `POST /payments/card` com `{ token, ..., saveCard: true }`
8. Backend salva o cartão em `SavedCard` via MP Customer API
9. Sucesso: navegar para `/client/creditos/sucesso`

### Fluxo C — Pagar sem salvar (menor destaque)

1. Cliente desmarcar o checkbox → CTA muda para "Pagar sem salvar"
2. Fluxo idêntico ao atual (`CardPaymentScreen` sem mudanças de persistência)
3. Backend: `POST /payments/card` com `{ token, ..., saveCard: false }`

### Fluxo D — Gerenciar cartões na SettingsScreen

1. Cliente acessa `/client/perfil` → seção "Cartões" visível
2. Para remover: toca em "Remover" → dialog de confirmação → confirma → `DELETE /users/me/cards/:id` → toast "Cartão removido."
3. Para definir padrão: toca em "Definir como padrão" → `PATCH /users/me/cards/:id` com `{ isDefault: true }` → toast "Cartão padrão atualizado."
