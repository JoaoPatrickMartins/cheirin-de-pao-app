## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP body → GrantCreditsSchema | Input não confiável do admin; quantity e reason validados por Zod antes de atingir a service |
| Service → Prisma $transaction | Operação financeira — deve ser atômica; falha parcial não pode deixar saldo inconsistente |
| Service → OneSignal API | Serviço externo; falha não pode bloquear a operação financeira principal |
| JWT → request.user.id | adminId extraído do JWT — nunca do body (T-10-02 equivalente ao padrão T-05-04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Spoofing | POST /admin/clients/:id/grant-credits | mitigate | Role check ADMIN no controller; adminId extraído do JWT (não do body) |
| T-10-02-02 | Tampering | GrantCreditsSchema quantity | mitigate | z.number().int().min(1) impede quantity=0 ou negativo; Zod parse no controller |
| T-10-02-03 | Tampering | reason enum | mitigate | z.enum(['Acerto','Bonificação','Compensação','Promoção']) — valores arbitrários rejeitados |
| T-10-02-04 | Repudiation | CreditTransaction ADMIN_GRANT | mitigate | adminId obrigatoriamente salvo no documento (auditoria); reason também salvo |
