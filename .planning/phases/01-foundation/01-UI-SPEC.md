# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Source of truth: `.projeto/design_handoff_cheirin_pao/README.md` + `app/brand.jsx`.

---

## Scope

Phase 1 delivers exactly one user-facing screen: **SplashScreen / InstallScreen** (route `/`). All other UI work in this phase is infrastructure (token definition in CSS, font loading, BreadMark component). This contract covers:

1. Global design token system (`globals.css @theme`) — foundational for all later phases
2. `BreadMark` SVG component (brand symbol)
3. `SplashScreen` — the only rendered page in Phase 1
4. PWA install prompt variants: Android native dialog trigger + iOS custom bottom sheet

## Spacing Scale

All values are multiples of 4. Phase 1 screen padding follows handoff exactly (20–24px).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon internal gaps, inline tight padding |
| sm | 8px | Compact element spacing, gap between icon + label |
| md | 16px | Default padding inside cards, gap between button stack items |
| lg | 24px | Screen horizontal padding (primary), section vertical gap |
| xl | 32px | Major vertical gap between splash sections |
| 2xl | 48px | App icon size clearance, large symbol display area |
| 3xl | 64px | Top/bottom inset for splash layout centering |

Exceptions:
- Screen horizontal padding: 20px (handoff spec), maps to between `md` and `lg` — use `px-5` (20px) in Tailwind v4
- Hit targets: minimum 44px height/width on ALL interactive elements (WCAG 2.5.5 / UI-10)
- App icon on splash: 132px square, border-radius 30%
- BreadMark symbol on splash: 86px
- App icon background: `#160C04` (darker than espresso, per handoff)
- Border radius on fields: 14px; buttons: 16px; cards: 18–22px; app icon: 30%; pills: 999px

Note: border-radius and SVG component size values (86px BreadMark, 132px app icon, 14px field border-radius, 18px card border-radius minimum) are design-exact values from brand.jsx — these govern shape rendering, not grid layout spacing, and are exempt from the 4px grid constraint.

### 60/30/10 Color Split

| Layer | Color | Usage |
|-------|-------|-------|
| 60% dominant | `#FAF5EC` (app-bg) | App background, all general surfaces |
| 30% secondary | `#FFFFFF` / `#FBF6EC` / `#F4EBDA` | Cards, install card, field backgrounds |
| 10% accent | `#E3AC3F` (gold) | Reserved for specific elements — see list below |

**Semantic role distinction:**
- `--color-gold` (`#E3AC3F`) = brand focal accent — logo, primary CTA, tagline; the dominant brand expression color
- `--color-accent` (`#B0702A`) = warm amber for secondary interactive text — ghost links, dismissal actions, focus states

**Accent (`#E3AC3F`) reserved exclusively for:**
- BreadMark SVG symbol color (default color prop)
- Primary CTA button on SplashScreen: "Instalar e criar conta" (gold variant button)
- Tagline text "PÃO FRESCO NA PORTA" on SplashScreen
- Switch toggle active state
- Focus border on OTP inputs (Phase 2; declared here for token consistency)
- Gold pill tone background `#F3DDA6` (not gold itself, but derived)

**Destructive:** No destructive actions in Phase 1. Token `--color-destructive` not needed this phase.

## Component Inventory (Phase 1 only)

### 1. BreadMark (`src/components/brand/BreadMark.tsx`)

SVG component. Paths extracted directly from `brand.jsx` (canonical source).

Props:
- `size`: number (default 100) — pixel dimension, both width and height
- `color`: string (default `#E3AC3F`)
- `reduced`: boolean (default false) — when true, hides lateral aroma waves and thickens stroke
- `side`: number (default 0.5) — opacity of lateral waves (full form only)
- `strong`: number (default 1) — opacity of central elements (full form only)

Usage rule: Pass `reduced={true}` when rendered at less than 48px. On SplashScreen: `size={86}`, `color="#E3AC3F"`, `reduced={false}`.

Accessibility: `role="img"` + `aria-label="Cheirin de Pão"` on the SVG element.

### 2. SplashScreen (`src/pages/splash/SplashScreen.tsx`)

Full viewport layout. Dark theme by exception (espresso background), not using `--color-app-bg`.

Primary focal point: app icon container (132px) with BreadMark symbol (86px) — the dominant visual element on the screen.

Layout (top to bottom, centered column):
1. Top vinheta: radial gradient from `rgba(227,172,63,0.15)` to transparent, centered at top of screen
2. App icon: 132px × 132px square, `border-radius: 30%`, background `#160C04`, box-shadow `shadow-strong`, contains `<BreadMark size={86} />`
3. App name: "Cheirin de Pão", 32px Bricolage Grotesque 700, color `#FBF3E4`, margin-top 20px
4. Tagline: "PÃO FRESCO NA PORTA", 12px Hanken Grotesk 700, letter-spacing 0.26em, color `#E3AC3F`, margin-top 8px
5. Spacer (flex-grow to push footer down)
6. Install card (white surface card, padding 20px, border-radius 22px, `shadow-soft`):
   - Card title: "Instalar o Cheirin", 15px weight 700
   - Primary CTA button (full-width, gold variant): "Instalar e criar conta"
   - Secondary link (ghost/text variant): "Já tenho conta — entrar"

Background: `#1E1207` (espresso), full viewport height.

### 3. Android Install Prompt (within SplashScreen)

When `isInstallable === true` AND `isStandalone === false`:
- The primary CTA "Instalar e criar conta" calls `triggerInstall()` from `useInstallPrompt` hook
- Button must have minimum height 44px and full width

### 4. iOS Install Bottom Sheet

When `isIOS === true` AND `isStandalone === false`:
- Renders a bottom sheet (fixed bottom, slides up on mount) over the SplashScreen
- Background: `#FFFFFF` surface card, border-radius 22px 22px 0 0, padding 24px
- Title: "Adicionar à tela inicial", 21px Bricolage Grotesque 700
- Step-by-step visual instructions (3 steps):
  1. Icon: share/arrow-up — "Toque em" + share icon glyph
  2. Icon: list — "Role e toque em"
  3. Text: "'Adicionar à Tela Inicial'"
- Each step row: icon 24px, text 15px Hanken Grotesk 400, gap 12px, min-height 44px
- Dismiss: tap outside or "Entendi" text button at bottom, 15px weight 700, color `#B0702A`
- Sheet uses `useInstallPrompt` hook: `isIOS` detection via UA regex

### 5. Icon System (`src/components/brand/Icon.tsx`)

Thin wrapper around the `Ic` path map from `brand.jsx`. Props: `name`, `size` (default 22), `stroke` (default 1.9), `color` (default `currentColor`). Renders a 24×24 viewBox SVG. All paths are stroke-based (no fill).

## Interaction Contract

### Transitions

| Element | Trigger | Property | Duration | Easing |
|---------|---------|----------|----------|--------|
| Button hover | mouseenter/mouseleave | `transform: translateY(-1px)` + `filter: brightness(1.05)` | 150ms | ease |
| Field focus | focus/blur | `border-color` from `--color-border` to `--color-accent` | 150ms | ease |
| iOS bottom sheet | mount | `transform: translateY(0)` from `translateY(100%)` | 300ms | ease-out |
| Progress bars | data load | `width` from 0 to target | 300ms | ease-out |

### Reduced Motion

Wrap all transitions in `@media (prefers-reduced-motion: reduce)` — set durations to 0ms. Do not remove transitions entirely (keep state changes readable).

### Hit Targets

Every interactive element in Phase 1 must satisfy:
- Minimum 44px height
- Minimum 44px width (or 44px touch area via padding if visually smaller)
- Applied via: `min-h-[44px]` or explicit `style={{ minHeight: 44 }}`

Affected elements in Phase 1: install button, secondary "Já tenho conta" link, iOS sheet dismiss, iOS sheet step rows.
