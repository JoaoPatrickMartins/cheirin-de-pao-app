<task type="auto" tdd="false">
  <name>Task 2: Script de migração standalone migrate-delivery-slots.ts</name>
  <files>apps/api/src/scripts/migrate-delivery-slots.ts</files>
  <action>
Criar o arquivo `apps/api/src/scripts/migrate-delivery-slots.ts`. O script usa `@prisma/client` diretamente (sem Fastify) para migrar dados no Atlas antes do deploy.

Lógica do script:

1. Conectar ao Prisma via `new PrismaClient()`.
2. Buscar todos os condomínios com `prisma.condominium.findMany()`.
3. Para cada condomínio, verificar se `deliverySlots` é um array vazio OU se contém strings (tipo legado). Detecção: `condo.deliverySlots.length === 0 || typeof condo.deliverySlots[0] === 'string'`.
4. Se precisa migrar: fazer `prisma.condominium.update({ where: { id: condo.id }, data: { deliverySlots: DEFAULT_SLOTS } })`.
5. Se já tem objetos com campo `name` (já migrado): pular sem alterar.
6. Ao final, imprimir: `Migração concluída: X condomínios atualizados, Y já estavam migrados`.

Constante DEFAULT_SLOTS (não mudar esses valores — per SLOT-02 e SLOT-06):
```
const DEFAULT_SLOTS = [
  { name: 'manha', time: '06:30', cutoffTime: '22:00', isActive: true },
  { name: 'tarde', time: '15:30', cutoffTime: '10:00', isActive: true },
]
```

O script termina com `prisma.$disconnect()` no bloco `finally`.

Adicionar em `package.json` de `apps/api` o script:
```
"migrate:slots": "tsx src/scripts/migrate-delivery-slots.ts"
```

NOTA PARA O EXECUTOR: Este script deve ser executado no ambiente de produção (com DATABASE_URL apontando para o Atlas) ANTES de fazer `prisma db push` e deploy da API. Registrar esse aviso no SUMMARY.
