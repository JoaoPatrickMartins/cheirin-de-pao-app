## Interaction Contract

### Multi-slot mode detection
- Condition: `slots.filter(s => s.isActive).length >= 2`
- When true: render multi-slot layout (2 sections)
- When false: render existing single-slot layout unchanged

### Slot section layout (multi-slot mode)
- Section "вҳҖпёҸ ManhГЈ В· 06:30": header pill + 7 day-rows with `StepperInline`
- Section "рҹҢҷ Tarde В· 15:30": header pill + 7 day-rows with `StepperInline`
- Each day-row: same structure as existing вҖ” label column (44px wide) + spacer + `StepperInline`
- Day-row opacity when quantity = 0: `0.66` (existing pattern)
- Day-row transition: `opacity 0.15s` (existing pattern)

### State initialization (multi-slot, new client)
- When schedule has no `days` field (null) but condo has 2 active slots: initialize `days` as `{ "06:30": DEFAULT_WEEKLY_QTY, "15:30": DEFAULT_WEEKLY_QTY }` where `DEFAULT_WEEKLY_QTY = { seg:0, ter:0, qua:0, qui:0, sex:0, sab:0, dom:0 }` вҖ” D-13
- No pre-population from legacy `weeklyQty` to avoid horГЎrio confusion

### Footer вҖ” multi-slot mode
- Remove delivery time from summary: show `"Consumo semanal"` label + `"X pГЈes"` value only
- Remove the `В· {deliveryTime}` suffix present in single-slot mode
- CTA button unchanged: `"Salvar agenda"` with check icon

### Footer вҖ” single-slot mode (unchanged)
- Keep existing: `"Consumo semanal"` + `"{X} pГЈes В· {deliveryTime}"`
- CTA unchanged

### Save behavior
- `saveSchedule(activeSlots)` вҖ” when `activeSlots.filter(s => s.isActive).length >= 2`: sends `{ days, notifyReconfigure }` to `PUT /schedules/me`
- Otherwise: sends `{ weeklyQty, deliveryTime, notifyReconfigure }` (legacy format)
- Success toast: `"Agenda salva!"` вҖ” identical copy to existing
- Error toast: `"NГЈo conseguimos salvar. Tente novamente."` вҖ” identical copy to existing
- Toast duration: 2500ms, position fixed top center, espresso background

### Loading state (multi-slot)
- Show 14 skeleton rows (7 per slot section) with 24px inter-section gap
- Same pulse animation as single-slot skeleton

### Toggle вҖ” notifyReconfigure
- No changes from existing implementation

### BannerCobertura
- No changes вҖ” receives `semana` (total sum) from hook
