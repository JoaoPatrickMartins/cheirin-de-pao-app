## Visual Hierarchy

**Primary visual anchor:** The slot section headers ("☀️ Manhã · 06:30" and "🌙 Tarde · 15:30") rendered in `--color-accent` with `--font-display` at weight 700 draw the eye before the day-rows. The creme pill background (`--color-surface-2`) frames each header and creates visual separation between the two slot sections. The 32px display font in the footer (stepper value when > 0) provides a secondary anchor at the bottom of the screen.

**Reading order:** AppBar title → slot section header (manhã) → day-rows (manhã) → slot section header (tarde) → day-rows (tarde) → sticky footer summary + CTA.

## Interaction Contract

### Multi-slot mode detection
- Condition: `slots.filter(s => s.isActive).length >= 2`
- When true: render multi-slot layout (2 sections)
- When false: render existing single-slot layout unchanged

### Slot section layout (multi-slot mode)
- Section "☀️ Manhã · 06:30": header pill + 7 day-rows with `StepperInline`
- Section "🌙 Tarde · 15:30": header pill + 7 day-rows with `StepperInline`
- Each day-row: same structure as existing — label column (44px wide) + spacer + `StepperInline`
- Day-row opacity when quantity = 0: `0.66` (existing pattern)
- Day-row transition: `opacity 0.15s` (existing pattern)

### State initialization (multi-slot, new client)
- When schedule has no `days` field (null) but condo has 2 active slots: initialize `days` as `{ "06:30": DEFAULT_WEEKLY_QTY, "15:30": DEFAULT_WEEKLY_QTY }` where `DEFAULT_WEEKLY_QTY = { seg:0, ter:0, qua:0, qui:0, sex:0, sab:0, dom:0 }` — D-13
- No pre-population from legacy `weeklyQty` to avoid horário confusion

### Footer — multi-slot mode
- Remove delivery time from summary: show `"Consumo semanal"` label + `"X pães"` value only
- Remove the `· {deliveryTime}` suffix present in single-slot mode
- CTA button unchanged: `"Salvar agenda"` with check icon

### Footer — single-slot mode (unchanged)
- Keep existing: `"Consumo semanal"` + `"{X} pães · {deliveryTime}"`
- CTA unchanged

### Save behavior
- `saveSchedule(activeSlots)` — when `activeSlots.filter(s => s.isActive).length >= 2`: sends `{ days, notifyReconfigure }` to `PUT /schedules/me`
- Otherwise: sends `{ weeklyQty, deliveryTime, notifyReconfigure }` (legacy format)
- Success toast: `"Agenda salva!"` — identical copy to existing
- Error toast: `"Não conseguimos salvar. Tente novamente."` — identical copy to existing
- Toast duration: 2500ms, position fixed top center, espresso background

### Loading state (multi-slot)
- Show 14 skeleton rows (7 per slot section) with 24px inter-section gap
- Same pulse animation as single-slot skeleton

### Toggle — notifyReconfigure
- No changes from existing implementation

### BannerCobertura
- No changes — receives `semana` (total sum) from hook
