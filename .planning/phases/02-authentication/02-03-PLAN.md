<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → AuthContext | Stored auth_token + auth_user are read on app mount; malformed JSON in auth_user would crash without a guard |
| ProtectedRoute → child component | Role check is client-side only — API also enforces auth on every request (defense in depth) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Spoofing | localStorage.auth_user tampered by XSS | accept | D-02 acknowledges XSS risk with localStorage. App does not render external HTML. Client-side role is for UX routing only — API enforces role on every protected request. |
| T-02-09 | Tampering | localStorage.auth_user JSON.parse | mitigate | Wrap JSON.parse(storedUser) in try/catch inside AuthContext useEffect. Malformed JSON clears state and user must re-login. |
| T-02-10 | Elevation of Privilege | ProtectedRoute role bypass via URL | accept | ProtectedRoute prevents wrong-role UI from rendering. API authenticate plugin is the security boundary — client-side guard is UX only. |
| T-02-11 | Information Disclosure | isLoading flash — brief window before redirect | accept | LoadingScreen renders during isLoading; child content never renders before auth check completes. |
</threat_model>
