## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API | apiFetch inclui JWT automaticamente via cookie/header; orderId vem da resposta da API (não da URL navegada pelo usuário) |
| Dialog UI → PATCH confirm | onConfirmed callback recebe orderId da resposta do GET /today — não de input do usuário |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-02 | Spoofing | CourierScreen (acesso sem login) | mitigate | CourierLayout já implementado desde Fase 2 faz role check CLIENT; requireCourier no backend (Plano 01) é a âncora — frontend é defense in depth |
