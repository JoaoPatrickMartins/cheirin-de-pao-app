<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JWT token → API | Token enviado pelo browser do entregador; payload decodificado mas userId/role vêm do JWT verificado — não do body |
| orderId (URL param) → service | ID de order fornecido pelo entregador via URL — pode apontar para order de outro entregador |
| body assign-courier → admin service | Admin envia courierId e lista de orderIds — pode tentar atribuir orders de outros contextos |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing / Elevation | PATCH /courier/orders/:id/confirm | mitigate | courier.service.confirmDelivery valida `order.courierId === courierId` (extraído do JWT) antes de chamar updateOrderStatus; retorna 403 se diferente |
| T-06-02 | Spoofing | GET /courier/orders/today e PATCH confirm | mitigate | preHandler `[fastify.authenticate, fastify.requireCourier]` em ambas as rotas; role != COURIER retorna 403 imediatamente |
| T-06-03 | Information Disclosure | GET /courier/orders/today | mitigate | Service filtra `courierId === request.user.id` via repository.findTodayByCourierId — orders sem courierId ou com courierId diferente nunca retornam |
| T-06-04 | Tampering | PATCH /admin/orders/assign-courier | mitigate | Role check ADMIN inline no controller.assignCourier; Zod valida courierId e orderIds; entregador não pode chamar este endpoint (preHandler authenticate verifica role separadamente) |
| T-06-05 | Tampering | Nominatim URL injection via endereço | accept | Endereço vem de prisma.condominium (banco) — não do entregador; `encodeURIComponent(address)` elimina risco residual |
| T-06-SC | Tampering | npm install react-leaflet leaflet @types/leaflet | mitigate | Package Legitimacy Audit em RESEARCH.md confirma todos [OK]; slopcheck 0.6.1 sem postinstall scripts |
</threat_model>
