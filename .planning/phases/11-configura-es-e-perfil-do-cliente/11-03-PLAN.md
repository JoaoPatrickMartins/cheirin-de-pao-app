## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → AuthUser | Dados de perfil persistidos; backward-compat com sessões antigas |
| LoginScreen → GET /client/profile | Fetch de perfil após login — falha não bloqueia autenticação |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-03-01 | Information Disclosure | AuthUser.cpf persiste no localStorage | accept | Dado já pertence ao usuário autenticado; localStorage é por origem (same-origin policy); risco aceito por ser dados do próprio usuário |
| T-11-03-02 | Tampering | updateUser sem validação do partial | accept | updateUser é interno ao AuthContext — só chamado por telas autenticadas do próprio usuário; sem exposição externa |
| T-11-03-03 | Spoofing | fetch GET /client/profile no login usa token recém-obtido | mitigate | apiFetch usa o token do localStorage que foi persistido por auth.login imediatamente antes — token válido garantido |
| T-11-03-SC | Tampering | npm installs | accept | Nenhum pacote novo neste plano |
</threat_model>
