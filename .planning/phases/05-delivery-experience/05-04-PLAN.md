## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → GET /orders/today (polling) | Requisição autenticada via cookie JWT; resposta contém dados do pedido do cliente |
| Browser → GET /notifications/me | Requisição autenticada; dados restritos ao usuário do JWT |
| Browser → PATCH /notifications/read-all | Sem payload malicioso; apenas operação de leitura confirmada |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Information Disclosure | useOrderTracking polling | accept | JWT enviado automaticamente via cookie httpOnly; endpoint /orders/today restringe por userId do JWT (enforçado no backend) |
| T-05-11 | Denial of Service | useOrderTracking polling | accept | Intervalo de 30s + cleanup no unmount; sem MAX_ATTEMPTS mas a request sai apenas quando a tela está montada — usuário que fechar a tela para o polling automaticamente |
| T-05-12 | Tampering | NotificationsScreen PATCH read-all | accept | PATCH não recebe payload — apenas marca como lidas; userId enforçado no backend via JWT |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Nenhum pacote novo — todos auditados em fases anteriores |
</threat_model>
