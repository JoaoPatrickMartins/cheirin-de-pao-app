## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Elevation of Privilege | /admin/suppliers, /admin/couriers, /admin/clients | mitigate | Role check `request.user?.role !== 'ADMIN'` inline primeiro statement de cada handler — 403 antes de qualquer acesso ao banco |
| T-07-03-02 | Tampering | POST /admin/couriers (criar entregador) | mitigate | Zod valida CPF com 11 dígitos; cpf=@unique no schema — erro 500 (constraint) se duplicado; adicionar catch de P2002 Prisma e retornar 409 |
| T-07-03-03 | Tampering | POST /admin/suppliers — isPrincipal | mitigate | Lógica de único principal: updateMany antes do create em transação implícita; sem race condition em prod (volume pequeno) |
| T-07-03-04 | Tampering | PATCH /admin/clients/:id/block — ID arbitrário | mitigate | Verificar role=CLIENT antes de toggle — admin não pode bloquear outro admin ou courier via esse endpoint |
| T-07-03-05 | Information Disclosure | GET /admin/clients/:id retorna Schedule + Orders | mitigate | Somente leitura; role check admin; dados de cliente só expostos a ADMIN autenticado |
| T-07-SC | Tampering | npm installs | accept | Nenhuma instalação nova neste plano |
