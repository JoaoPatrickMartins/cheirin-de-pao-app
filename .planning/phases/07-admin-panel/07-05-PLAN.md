## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Apenas ADMIN pode ver financeiro e processar estornos |
| API → Mercado Pago | Chave MP_ACCESS_TOKEN no servidor — nunca exposta ao cliente |
| API → MongoDB | $runCommandRaw para agregacao — sem input de usuario no pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05-01 | Elevation of Privilege | /admin/financial e /admin/payments | mitigate | Role check ADMIN inline no controller — 403 antes de qualquer operacao |
| T-07-05-02 | Tampering | POST /admin/payments/:id/refund — estorno fraudulento | mitigate | Verificar Payment.status === 'PAID' antes de chamar MP; idempotencia garantida pela API MP; $transaction atomica evita debito duplo |
| T-07-05-03 | Tampering | POST /admin/payments/:id/refund — ID invalido | mitigate | findUnique com 404 antes de chamar MP — nunca chamar refundApi sem Payment existente no banco |
| T-07-05-04 | Information Disclosure | GET /admin/financial expoe dados de receita | mitigate | Endpoint exige autenticacao ADMIN + role check; dados financeiros nunca expostos sem token valido |
| T-07-05-05 | Denial of Service | $runCommandRaw sem limite | mitigate | Sempre incluir $match com date range antes do $lookup — nao agregar collection inteira |
| T-07-05-06 | Tampering | Estorno parcial via customQuantity manipulado | mitigate | creditsToDebit calculado no server (D-04: estorno TOTAL) — cliente nao envia quantidade de creditos a debitar |
| T-07-SC | Tampering | npm installs | accept | mercadopago ja instalado no projeto |
